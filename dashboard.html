CCS Code

An AI-powered personal knowledge base CLI. Sync from GitHub, harvest local IDE histories, ingest files into a cross-linked wiki, visualize it as an interactive graph, and answer questions from your own compounding knowledge.

Terminal-first Plain Markdown vault LLM-enriched wiki Migration intelligence

What It Does

You run ccs-code in your terminal. A set of composable commands form a pipeline. Each time you run through it your knowledge base grows and compounds — conversations from Claude, Cursor, and VS Code become searchable wiki pages linked by AI-generated wikilinks and tags.

pipeline 
# Sources
GitHub repos  +  IDE histories (Cursor, VS Code, Claude)  +  local files
        │
        ▼ /sync + /harvest + drop into raw/
        │
        ▼ /ingest  →  wiki/ pages (Markdown, auto-merged)
        │
        ▼ /enrich  →  AI summaries + tags + [[wikilinks]]
        │
        ├── /graph   →  interactive vis.js graph in browser
        └── /ask     →  TF-IDF search → LLM answers with citations

Quick Start

1. Initialize your vault

/vault init                          # creates vault at ./vault
/vault init ~/my-knowledge-base      # custom location

2. Add sources to ccs.yaml

yaml 
sources:
  - type: github
    repos:
      - my-org/auth-svc
      - my-org/payment-svc
    include: [commits, prs, issues, readme]
    token_env: GH_TOKEN

llm:
  provider: anthropic
  model: claude-sonnet-4-6

3. Run the pipeline

/sync       # pull GitHub sources
/harvest    # mine local AI chat histories
/ingest     # convert raw files → wiki pages
/enrich     # AI summaries + tags + wikilinks
/graph      # open interactive knowledge graph
/ask "what did I work on last week?"
💡
Tip — type / for commands

In the terminal UI, type / for the full command menu, ? for keyboard shortcuts, and ↑ to retrieve a queued prompt.

Pipeline

The seven stages form a compounding pipeline. You can run them independently or chain them together.

1
Vault Init /vault init [path]
Creates the folder structure and saves the vault path to ~/.ccs/config.json. All other commands auto-discover the active vault.
~/.ccs/config.json one-time setup
2
Sync /sync
Pulls commits, PRs, issues, and README from GitHub repos listed in ccs.yaml. Writes raw files to raw/github/. Uses GitHub REST API — no local clone needed.
GitHub REST API raw/github/ GH_TOKEN
3
Harvest /harvest
Extreme Scraper logic for the AI era. Mines conversation history from Cursor generations, VS Code Copilot .jsonl sessions, Claude Code project logs, and Antigravity memories. Writes one file per memory to raw/memories/.
Cursor VS Code Copilot Claude Code Antigravity raw/memories/
4
Ingest /ingest
Reads everything in raw/ and converts it to Markdown wiki pages in wiki/. Re-running merges new content — never overwrites. HTML → wiki/concepts/, conversations.json → wiki/conversations/.
HTML parser Markdown merge YAML frontmatter
5
Enrich /enrich
Sends each un-enriched wiki page to the LLM. Gets back a one-paragraph summary, keyword tags, and related page titles. Injects summary and tags into frontmatter. Appends ## Related with [[wikilinks]]. Safe to re-run — skips already-enriched pages.
LLM per-page [[wikilinks]] frontmatter injection idempotent
6
Graph /graph
Builds an interactive vis.js knowledge graph. Node size scales with connections. Colors group by topic (code=purple, design=pink, ai=green, data=yellow…). Hover fades unconnected nodes. Click opens info panel. Search filters live.
vis.js TF-IDF keyword edges Obsidian-style glow output/graph.html
7
Ask /ask <question>
TF-IDF keyword search across all wiki pages. Top 6 matching pages injected into LLM context. LLM answers with [[page-title]] citations. Title token matches score 5×, tag matches 3×, body term frequency 1 + log(freq).
TF-IDF retrieval RAG [[citations]] top-6 context

Vault Structure

The vault is plain Markdown — any AI tool that reads files can use it directly.

vault/ ├── ccs.yaml ← sources + LLM provider config ├── raw/ ← sync inbox (never edit manually) │ ├── github/ ← /sync writes here │ ├── memories/ ← /harvest writes here │ └── uploads/ ← drop local files here ├── wiki/ ← auto-maintained by /ingest + /enrich │ ├── _master-index.md ← /index rebuilds this │ ├── concepts/ ← from HTML + text │ └── conversations/ ← from conversations.json ├── output/ │ ├── graph.html ← /graph output │ └── rewrite-context/ ├── skills/ │ ├── wiki-ingest/SKILL.md │ ├── wiki-query/SKILL.md │ ├── graph-build/SKILL.md │ └── rewrite-plan/SKILL.md └── .claude-plugin/plugin.json ← Claude Code manifest ~/.ccs/ ├── config.json ← global: stores activeVault path └── guide.html ← /guide output

Wiki page frontmatter

Every ingested page gets standard frontmatter. Enrich adds summary and tags.

yaml 
---
title:       auth-svc
type:        concept
source:      raw/github/auth-svc-readme.html
last_synced: 2026-04-22
staleness:   fresh
summary:     "JWT-based authentication service used by all frontend clients"
tags:        [auth, jwt, security, backend]
---

Command Reference

All commands are available in the terminal UI via the / menu. Type / to open the command picker.

/vault

Manages the active vault — creation, status, and health checks.

Command Description Example
/vault init [path] Create a new vault. Saves path to ~/.ccs/config.json. All other commands pick it up automatically. /vault init ~/my-kb
/vault status Show active vault path and current counts: wiki pages, raw files ready to ingest, skills loaded. /vault status
/vault audit Health check: compares harvested memory count against ingested wiki pages. Flags gaps in coverage. /vault audit

/sync

Pulls the latest commits, PRs, issues, and README from GitHub repos configured in ccs.yaml.

Command Description Example
/sync Sync all configured sources. Writes raw files to raw/github/. Uses GH_TOKEN env var. /sync
⚙️
Requires GH_TOKEN

Set GH_TOKEN or GITHUB_TOKEN in your environment. Add repos to sources: in ccs.yaml.

/harvest

Mines conversation history from your local development environment. No manual export needed.

Source What it reads Output
Cursor Generations (prompt logs) + composer histories from local SQLite DB raw/memories/cursor-*
VS Code GitHub Copilot chat sessions stored as .jsonl files raw/memories/vscode-*
Claude Code Project-specific session logs from Claude Code CLI raw/memories/claude-*
Antigravity Local brain memories from the Antigravity tool raw/memories/antigravity-*

/ingest

Converts everything in raw/ to Markdown wiki pages. Re-running is safe — new content is merged, existing pages are never overwritten.

Command Description Example
/ingest Process all raw files. HTML → wiki/concepts/, conversations → wiki/conversations/. Merges into existing pages. /ingest

/enrich

AI-enriches every wiki page that doesn't already have a summary: field. Makes one LLM call per page — runs best overnight on large vaults.

summary
One-paragraph plain-language description of what the page covers, injected into frontmatter.
tags
3–6 keyword tags added to frontmatter for search and graph grouping.
relatedTitles
2–5 exact page titles from the same vault that are semantically related. Used to build [[wikilinks]] in the ## Related section.
⚡
Progress tracking

The tool card shows live progress: 47/180 · 26% · react-hooks-2024. Press Esc to cancel — your queued prompt will be restored to the input box.

/graph

Builds a self-contained output/graph.html and opens it in the browser. All rendering is client-side — no server needed.

Node size
7 + min(22, connections × 2.5) — hubs are visually prominent.
Node color
Topic group — code=purple, design=pink, ai=green, data=yellow, devops=blue. Glow shadow per color (Obsidian-style).
Edge building
TF-IDF keywords from title + summary. Keywords appearing in 2–30 nodes create edges. Max 12 connections per node.
Interactions
Hover → fade unconnected nodes. Click → info panel. Search bar → live node filter. Legend click → isolate topic group.

/ask

Retrieval-augmented search over your entire wiki. Fast and runs offline (TF-IDF) before sending the top matches to the LLM.

Command Description Example
/ask <question> Search wiki, inject top 6 pages into LLM context, return answer with [[citations]]. /ask what does auth-svc depend on?
Title token match
Score multiplier: 5×
Tag match
Score multiplier: 3×
Body frequency
1 + log(freq), normalized by query length
Context window
Top 6 scoring pages injected. LLM answers with [[page-title]] inline citations.

/migrate

Migration intelligence — analyze legacy codebases and generate verified migration contracts and per-component context docs for AI agents.

Command Description Example
/migrate rewrite Full pipeline: fetch repo → detect framework → analyze components → verify claims → write artifacts + dashboard. /migrate rewrite --repo <url> --to csharp --yes
/migrate scan Scan for SOAP/WCF external service calls. Falls back to general code analysis if none found. /migrate scan --repo <url> --lang csharp --yes
/migrate reverse-eng Persist reverse-engineering artifacts only — no agent migration contract. Good for documentation. /migrate reverse-eng --repo <url> --yes
/migrate status Component-by-component migration progress table. /migrate status
/migrate open Open the latest run folder in Finder/Explorer, or open its dashboard.html. /migrate open --dashboard
/migrate verify Mark a component as human-verified after reviewing its context doc. /migrate verify AuthService --by max
/migrate db Live database schema extraction. Read-only, user-approved, credentials never stored. /migrate db --service UserService --yes
/migrate clean Remove old run folders to keep the migration root tidy. /migrate clean --all --yes

migrate rewrite — flags

--repo <url>
GitHub URL of the source repository. Required.
--to <language>
Target language: csharp, typescript, python, java, go. Default: python.
--from <framework>
Override auto-detected source framework: aspnet-core, spring-boot, express.
--context <path>
Load a business/architecture doc to anchor LLM decisions. Repeatable. Copied into architecture-context/ for run reproducibility.
--no-context
Disable all context docs, including auto-discovered workspace docs.
--yes
Skip cost confirmation gate. Without this flag, the command shows a token estimate and waits.

Configuration — ccs.yaml

Place ccs.yaml at the root of your vault. It controls sources and the LLM provider.

yaml 
# Sources to sync
sources:
  - type: github
    repos:
      - my-org/auth-svc
      - my-org/payment-svc
    include: [commits, prs, issues, readme]
    token_env: GH_TOKEN

# LLM provider for /enrich and /ask
llm:
  provider: anthropic
  model: claude-sonnet-4-6

LLM Providers

Configure your provider in ccs.yaml and set the required environment variables.

Anthropic
ANTHROPIC_API_KEY
Claude Sonnet, Opus, Haiku. Best for enrichment and migration analysis.
provider: anthropic
model: claude-sonnet-4-6
OpenAI
OPENAI_API_KEY
GPT-4o and compatible models. Full feature parity.
provider: openai
model: gpt-4o
Enterprise
CCS_ENTERPRISE_*
Azure OpenAI with OAuth 2.0 client credentials. Five env vars required.
provider: enterprise

Enterprise environment variables

CCS_ENTERPRISE_CLIENT_ID
OAuth client ID for the Azure AD application
CCS_ENTERPRISE_CLIENT_SECRET
OAuth client secret
CCS_ENTERPRISE_AUTH_URL
Token endpoint URL (e.g. https://login.microsoftonline.com/<tenant>/oauth2/v2.0/token)
CCS_ENTERPRISE_SCOPE
OAuth scope string
CCS_ENTERPRISE_API_BASE
Azure OpenAI endpoint base URL

Integration with AI Tools

The vault is plain Markdown. Any AI tool that reads files can index it and use it as live context.

Claude Code
Point at the vault directory — CLAUDE.md + skills/ auto-load as context for the agent.
GitHub Copilot
Open vault as a VS Code workspace — wiki pages appear as context in Copilot Chat.
Cursor / Windsurf
Use @filename to reference any wiki page in chat. Point the AI rules at wiki/.
Any vector agent
Index wiki/ in your vector store — YAML frontmatter fields map directly to metadata.
